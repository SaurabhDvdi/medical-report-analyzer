================================================================================
                    MEDICAL REPORT ANALYZER - PROJECT SUMMARY
================================================================================

PROJECT OVERVIEW
----------------
A comprehensive full-stack application for uploading, analyzing, and visualizing 
medical reports. The system features a Python/FastAPI backend for all data 
processing, analytics, and visualization, and a React frontend for the user 
interface.

================================================================================
                            BACKEND (Python/FastAPI)
================================================================================

DATABASE MODELS (SQLAlchemy ORM with SQLite)
--------------------------------------------
1. Users
   - id, email, password_hash, full_name, role (patient/doctor)
   - created_at timestamp
   - Relationships: reports, medicines, doctor_notes

2. Reports
   - id, user_id, file_name, file_path, file_type
   - upload_date, ocr_status (pending/processing/completed/failed)
   - extracted_text, ai_summary
   - category_id (foreign key to ReportCategories)
   - Relationships: user, category, lab_values, doctor_notes

3. LabValues
   - id, report_id, parameter_name, value (float)
   - unit, reference_range, is_abnormal (boolean)
   - Optimized for pandas DataFrame operations
   - Relationships: report

4. Medicines
   - id, user_id, name, dosage, frequency
   - start_date, end_date, status (current/past)
   - created_at timestamp
   - Relationships: user

5. DoctorNotes
   - id, doctor_id, patient_id, report_id (nullable)
   - note_text (rich text), created_at
   - Relationships: doctor, patient, report

6. ReportCategories
   - id, name (unique), description
   - created_at timestamp
   - Dynamic expansion as new report types are detected
   - Relationships: reports

AUTHENTICATION SYSTEM
---------------------
- JWT-based authentication with HTTPBearer
- Password hashing using bcrypt (passlib)
- Token expiration: 30 minutes
- Role-based access control (patient/doctor)
- Endpoints:
  * POST /api/auth/register - Register new user
  * POST /api/auth/login - Login user

SERVICES
--------

1. OCR Service (services/ocr_service.py)
   - Supports Tesseract and EasyOCR
   - Handles PDF and image files (PNG, JPG, JPEG)
   - Automatic fallback if EasyOCR unavailable
   - Background processing after file upload

2. NLP Service (services/nlp_service.py)
   - Uses Hugging Face Transformers
   - Primary model: facebook/bart-large-cnn
   - Fallback model: t5-small
   - Generates AI summaries of medical reports
   - Runs completely locally (no external APIs)

3. Analytics Service (services/analytics_service.py)
   - Pandas for data manipulation
   - NumPy for numerical operations
   - Matplotlib/Seaborn for visualization
   - Generates charts as PNG files:
     * Trend analysis (time-series line charts)
     * Parameter comparisons (bar charts)
     * Health summaries (pie charts + bar charts)
     * Correlation heatmaps
   - All charts generated server-side

4. Report Parser (services/report_parser.py)
   - Extracts structured lab values from OCR text
   - Supports 17+ common medical parameters:
     * Hemoglobin, RBC, WBC, Platelet
     * Glucose, Creatinine, Urea
     * Cholesterol, HDL, LDL, Triglycerides
     * ALT, AST, Bilirubin
     * TSH, T3, T4
   - Automatic abnormality detection based on reference ranges
   - Dynamic category detection (Blood Test, Lipid Profile, Liver Function, etc.)

API ENDPOINTS
-------------

Report Management:
- POST /api/reports/upload - Upload medical report (with background processing)
- GET /api/reports - Get all reports (filtered by user role)
- GET /api/reports/{id} - Get report details with lab values
- DELETE /api/reports/{id} - Delete report and associated data
- GET /api/reports/{id}/download - Download original report file

Lab Values:
- GET /api/lab-values - Get lab values with filters:
  * parameter_name
  * start_date, end_date
  * Role-based filtering

Medicine Management:
- POST /api/medicines - Add new medicine
- GET /api/medicines - Get all medicines for user
- PUT /api/medicines/{id} - Update medicine
- DELETE /api/medicines/{id} - Delete medicine

Analytics:
- GET /api/analytics/trend/{parameter_name} - Get trend chart (PNG)
- GET /api/analytics/comparison?parameter_names=param1,param2 - Comparison chart
- GET /api/analytics/health-summary - Health summary visualization
- GET /api/analytics/correlation - Correlation heatmap

Doctor Notes:
- POST /api/doctor-notes - Create doctor note (doctors only)
- GET /api/doctor-notes - Get notes (filtered by patient/report)

Export:
- GET /api/export/csv - Export lab values as CSV

Other:
- GET /api/report-categories - Report OCR / type categories (legacy path was /api/categories)
- GET /api/categories - Doctor clinical categories (discovery & registration)
- GET /api/specialties?category_id= - Specialties for a doctor category
- POST /api/specialties - Doctors create an additional specialty under a category
- GET /api/doctors?name=&category_id=&specialty_id= - Doctor discovery (auth required)
- GET /api/patient/discovery-stats - Total doctors on platform + patient’s accepted doctor count
- GET /api/doctor/assignment-stats - Total patients on platform + doctor’s assigned (accepted) count
- POST /api/patient/doctor-access - Patient requests access to a doctor
- GET /api/patient/doctor-access - Patient’s access rows
- GET /api/doctor/patient-access-requests?status=pending - Doctor incoming requests
- POST /api/doctor/patient-access-requests/{id}/accept | /reject
- GET /api/users/patients - Get all patients (doctors only)

FILE STORAGE
------------
- Upload directory: backend/uploads/
- Chart directory: backend/charts/
- All files stored locally
- Automatic cleanup on report deletion

APRIL 17, 2026 - PRODUCTION-GRADE ARCHITECTURE UPGRADE
================================================================================

MAJOR ARCHITECTURAL TRANSFORMATION: From basic UI/API coupling to production-grade,
scalable, analytics-ready system architecture.

--- OBJECTIVES ACHIEVED ---

1. SERVICE LAYER INTRODUCTION (Decoupled UI from API)
2. REACT QUERY INTEGRATION (Data fetching & caching)
3. N+1 API CALLS ELIMINATION (Critical performance fix)
4. ROLE-BASED ROUTE PROTECTION (Security hardening)
5. STANDARDIZED ERROR HANDLING (Toast-based notifications)
6. ANALYTICS LAYER PREPARATION (Future interactive charts ready)
7. BACKEND ENDPOINT OPTIMIZATION (Server-side aggregation)
8. SECURITY HARDENING (No API calls without auth)
9. CODE CLEANUP (Removed redundant states, direct API calls)

--- NEW ARCHITECTURE OVERVIEW ---

BEFORE: UI → Direct API Calls → Backend (Tight coupling, N+1 problems)
AFTER:  UI → Service Layer → React Query → Backend (Loose coupling, optimized)

--- SERVICE LAYER CREATED ---

Location: frontend/src/services/
Files Created:
- analyticsService.js: Health summary, correlation data
- reportService.js: Reports, report details, reports summary
- userService.js: All user/auth operations

Pattern:
export const getHealthSummary = () => api.get('/api/analytics/health-summary', { responseType: 'blob' })
export const getReports = () => api.get('/api/reports')

Rule: NO component calls api.get directly anymore. ALL API calls through services.

--- REACT QUERY INTEGRATION ---

Package: @tanstack/react-query
Configuration:
- QueryClientProvider wrapped around App
- Default staleTime: 5 minutes
- Default cacheTime: 10 minutes
- Automatic background refetching

Refactor Pattern:
BEFORE: useState + useEffect + multiple API calls
AFTER:  useQuery({ queryKey: ['reports'], queryFn: getReports })

--- N+1 API CALLS FIXED (CRITICAL PERFORMANCE IMPROVEMENT) ---

PROBLEM: PatientDashboard looped through reports making individual API calls:
for (const report of reports) {
  const detail = await api.get(`/api/reports/${report.id}`)  // N+1 calls!
}

SOLUTION: New aggregated backend endpoint + single frontend call

Backend: GET /api/reports/summary
- Returns: total_reports, abnormal_count, recent_reports
- Server-side aggregation (no N+1 queries)
- Role-based filtering (patient/doctor scope)

Frontend: Single optimized call replaces loop
const { data: reportsSummary } = useQuery({
  queryKey: ['reports-summary'],
  queryFn: getReportsSummary
})

Performance Impact: From O(n) API calls to O(1) API call

--- ROLE-BASED ROUTE PROTECTION ---

Component: RoleRoute.jsx (frontend/src/components/RoleRoute.jsx)
Logic:
- Blocks rendering until auth determined (loading = null)
- Redirects unauthorized users to /dashboard
- Applied to analytics routes (patient-only) and doctor routes (doctor-only)

Usage:
<Route path="analytics/health-summary" element={
  <RoleRoute requiredRole="patient">
    <HealthSummaryPage />
  </RoleRoute>
} />

--- STANDARDIZED ERROR HANDLING ---

Rules:
- API errors → toast notifications (react-hot-toast)
- Validation → inline feedback
- Fatal errors → fallback UI

Pattern:
BEFORE: console.error('Error:', err)
AFTER:  showToast(error.response?.data?.detail || 'Failed to load', 'error')

Structure Created: frontend/src/analytics/
- components/AnalyticsCharts.jsx: Placeholder for future interactive charts
- hooks/useAnalytics.js: Data fetching hooks for structured analytics

Prepared for:
- JSON-based data (vs current blob images)

- Real-time analytics
- Advanced visualizations

--- BACKEND OPTIMIZATION ---

New Endpoint: GET /api/reports/summary
- Aggregated calculations server-side
- No repeated queries
- Efficient joins and filtering

--- SECURITY HARDENING ---

Frontend:
- AuthContext blocks rendering until auth determined

Backend:

--- EXAMPLE REFACTORED COMPONENT: PatientDashboard ---

BEFORE (N+1 calls, direct API, useState/useEffect):
const [stats, setStats] = useState({...})
useEffect(() => {
  const reports = await api.get('/api/reports')  // Direct API call
  for (const report of reports) {
    const detail = await api.get(`/api/reports/${report.id}`)  // N+1!
  }
}, [])

AFTER (React Query, Service Layer, Aggregated Data):
const { data: reportsSummary } = useQuery({
  queryKey: ['reports-summary'],
  queryFn: getReportsSummary  // Through service layer
})

const stats = {
  totalReports: reportsSummary?.total_reports || 0,
  abnormalValues: reportsSummary?.abnormal_count || 0,  // Pre-calculated
  recentReports: reportsSummary?.recent_reports || [],
}
