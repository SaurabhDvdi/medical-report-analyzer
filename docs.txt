================================================================================
                    MEDICAL REPORT ANALYZER - PROJECT SUMMARY
================================================================================

PROJECT OVERVIEW
----------------
A comprehensive full-stack application for uploading, analyzing, and visualizing 
medical reports. The system features a Python/FastAPI backend for all data 
processing, analytics, and visualization, and a React frontend for the user 
interface.

================================================================================
                            BACKEND (Python/FastAPI)
================================================================================

DATABASE MODELS (SQLAlchemy ORM with SQLite)
--------------------------------------------
1. Users
   - id, email, password_hash, full_name, role (patient/doctor)
   - created_at timestamp
   - Relationships: reports, medicines, doctor_notes

2. Reports
   - id, user_id, file_name, file_path, file_type
   - upload_date, ocr_status (pending/processing/completed/failed)
   - extracted_text, ai_summary
   - category_id (foreign key to ReportCategories)
   - Relationships: user, category, lab_values, doctor_notes

3. LabValues
   - id, report_id, parameter_name, value (float)
   - unit, reference_range, is_abnormal (boolean)
   - Optimized for pandas DataFrame operations
   - Relationships: report

4. Medicines
   - id, user_id, name, dosage, frequency
   - start_date, end_date, status (current/past)
   - created_at timestamp
   - Relationships: user

5. DoctorNotes
   - id, doctor_id, patient_id, report_id (nullable)
   - note_text (rich text), created_at
   - Relationships: doctor, patient, report

6. ReportCategories
   - id, name (unique), description
   - created_at timestamp
   - Dynamic expansion as new report types are detected
   - Relationships: reports

AUTHENTICATION SYSTEM
---------------------
- JWT-based authentication with HTTPBearer
- Password hashing using bcrypt (passlib)
- Token expiration: 30 minutes
- Role-based access control (patient/doctor)
- Endpoints:
  * POST /api/auth/register - Register new user
  * POST /api/auth/login - Login user

SERVICES
--------

1. OCR Service (services/ocr_service.py)
   - Supports Tesseract and EasyOCR
   - Handles PDF and image files (PNG, JPG, JPEG)
   - Automatic fallback if EasyOCR unavailable
   - Background processing after file upload

2. NLP Service (services/nlp_service.py)
   - Uses Hugging Face Transformers
   - Primary model: facebook/bart-large-cnn
   - Fallback model: t5-small
   - Generates AI summaries of medical reports
   - Runs completely locally (no external APIs)

3. Analytics Service (services/analytics_service.py)
   - Pandas for data manipulation
   - NumPy for numerical operations
   - Matplotlib/Seaborn for visualization
   - Generates charts as PNG files:
     * Trend analysis (time-series line charts)
     * Parameter comparisons (bar charts)
     * Health summaries (pie charts + bar charts)
     * Correlation heatmaps
   - All charts generated server-side

4. Report Parser (services/report_parser.py)
   - Extracts structured lab values from OCR text
   - Supports 17+ common medical parameters:
     * Hemoglobin, RBC, WBC, Platelet
     * Glucose, Creatinine, Urea
     * Cholesterol, HDL, LDL, Triglycerides
     * ALT, AST, Bilirubin
     * TSH, T3, T4
   - Automatic abnormality detection based on reference ranges
   - Dynamic category detection (Blood Test, Lipid Profile, Liver Function, etc.)

API ENDPOINTS
-------------

Report Management:
- POST /api/reports/upload - Upload medical report (with background processing)
- GET /api/reports - Get all reports (filtered by user role)
- GET /api/reports/{id} - Get report details with lab values
- DELETE /api/reports/{id} - Delete report and associated data
- GET /api/reports/{id}/download - Download original report file

Lab Values:
- GET /api/lab-values - Get lab values with filters:
  * parameter_name
  * start_date, end_date
  * Role-based filtering

Medicine Management:
- POST /api/medicines - Add new medicine
- GET /api/medicines - Get all medicines for user
- PUT /api/medicines/{id} - Update medicine
- DELETE /api/medicines/{id} - Delete medicine

Analytics:
- GET /api/analytics/trend/{parameter_name} - Get trend chart (PNG)
- GET /api/analytics/comparison?parameter_names=param1,param2 - Comparison chart
- GET /api/analytics/health-summary - Health summary visualization
- GET /api/analytics/correlation - Correlation heatmap

Doctor Notes:
- POST /api/doctor-notes - Create doctor note (doctors only)
- GET /api/doctor-notes - Get notes (filtered by patient/report)

Export:
- GET /api/export/csv - Export lab values as CSV

Other:
- GET /api/categories - Get all report categories
- GET /api/users/patients - Get all patients (doctors only)

FILE STORAGE
------------
- Upload directory: backend/uploads/
- Chart directory: backend/charts/
- All files stored locally
- Automatic cleanup on report deletion

================================================================================
                            ARCHITECTURE DECISIONS
================================================================================

1. SEPARATION OF CONCERNS
   - Backend handles ALL data processing, analytics, and visualization
   - Frontend is purely presentational
   - Clear API contract between layers

2. DATABASE DESIGN
   - SQLite for simplicity and local deployment
   - SQLAlchemy ORM for type safety and migrations
   - Optimized schema for pandas operations
   - Foreign key relationships for data integrity

3. AUTHENTICATION
   - JWT tokens for stateless authentication
   - Bcrypt for secure password hashing
   - Role-based access control at API level

4. FILE PROCESSING
   - Background tasks for non-blocking OCR
   - Local file storage (no cloud dependencies)
   - Automatic cleanup on deletion

5. ANALYTICS
   - Server-side chart generation
   - PNG images served to frontend
   - Efficient pandas operations
   - Cached chart generation

6. ERROR HANDLING
   - Graceful degradation
   - User-friendly error messages
   - Comprehensive logging

7. SCALABILITY
   - Modular service architecture
   - Easy to add new report types
   - Extensible analytics pipeline
   - ML model integration ready
